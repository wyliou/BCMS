## 1. 系統簡介

BCMS 是企業年度預算蒐集平台。系統管理員開啟預算週期後，平台自動產生各部門
Excel 預算樣板，部門經理人下載並上傳已填寫的預算，人資及財務部門匯入人力預算
與公攤費用，財務管理員透過彙整報表監控預算與實績比較。

### 操作流程總覽

```
建立週期 → 開放週期（產生樣板 + 發送通知）
    → 各部門下載樣板 → 填寫 → 上傳預算
    → 人資匯入人力預算
    → 財務匯入公攤費用 / 匯入實績
    → 彙整報表檢視 → 匯出報表
    → 關閉週期
```

### 系統導覽列

登入後，左側導覽列依角色顯示可存取的頁面：

| 功能 | 說明 |
|------|------|
| 儀表板 | 填報狀態總覽 |
| 彙整報表 | 多來源合併預算報表 |
| 上傳預算 | 部門預算檔案上傳 |
| 人力預算匯入 | 人力預算資料匯入 |
| 公攤費用匯入 | 公攤費用資料匯入 |
| 週期管理 | 預算週期建立與管理 |
| 會科主檔 | 會計科目維護 |
| 組織架構 | 組織單位管理 |
| 使用者管理 | 使用者帳號與角色管理 |
| 稽核日誌 | 系統操作紀錄查詢 |

## 2. 角色與權限

系統採用角色型存取控制 (RBAC)，每位使用者可擁有一或多個角色：

| 角色 | 中文 | 範圍 | 說明 |
|------|------|------|------|
| SystemAdmin | 系統管理員 | 全域 | 完整系統管理權限：使用者管理、組織架構、週期管理、重開週期 |
| FinanceAdmin | 財務管理員 | 全域 | 儀表板、彙整報表、公攤費用匯入、週期管理、會科主檔 |

## 3. 登入與登出

### 登入

1. 開啟瀏覽器，連線至系統首頁
2. 點擊 **「SSO 登入」** 按鈕
3. 系統將導向企業單一登入 (SSO) 頁面
4. 使用企業帳號密碼完成驗證
5. 驗證成功後自動跳轉至儀表板（或依角色導向首頁）

> 若使用者尚未被指派角色，登入後將看到 **「無存取權限」** 提示，請聯繫系統管理員。

### 登出

點擊右上角的 **「登出」** 按鈕即可登出系統。

### 閒置逾時

系統預設閒置 30 分鐘後自動登出。單次登入最長有效時間為 8 小時，超過後需重新登入。

## 4. 儀表板

**路徑：** 儀表板  
**可用角色：** SystemAdmin、FinanceAdmin、UplineReviewer、CompanyReviewer

儀表板提供當前預算週期的填報進度即時監控。

### 4.1 選擇週期

頁面頂部有 **「選擇週期」** 下拉選單，系統自動選取最新開放中的週期。可手動切換至其他週期。

### 4.2 摘要卡片

四張摘要卡片顯示當前週期概況：

| 卡片 | 說明 |
|------|------|
| 填報單位總數 | 本週期所有需填報的組織單位數 |
| 已上傳 | 已完成至少一次預算上傳的單位數 |
| 未下載 | 尚未下載預算樣板的單位數 |
| 已通知重傳 | 已發送重傳通知的單位數 |

### 4.3 填報狀態表格

顯示每個填報單位的詳細狀態：

| 欄位 | 說明 |
|------|------|
| 單位代碼 | 組織單位代碼 |
| 單位名稱 | 組織單位名稱 |
| 狀態 | 未下載 / 已下載 / 已上傳 / 已通知重傳 |
| 最後上傳時間 | 最近一次上傳的時間戳記 |
| 版本 | 目前上傳版本編號 |
| 操作 | 通知重傳按鈕（限 FinanceAdmin / UplineReviewer） |

可使用 **狀態篩選** 及 **單位搜尋** 縮小檢視範圍。

> 儀表板每 5 秒自動更新資料。若資料更新較慢，頁面會顯示提示訊息。

### 4.4 通知重傳

1. 在填報狀態表格中，點擊目標單位的 **「通知重傳」** 按鈕
2. 在彈出視窗中填寫 **重傳原因**（必填）
3. 可選填 **目標版本** 編號
4. 點擊 **「送出通知」**
5. 系統將發送 Email 通知該單位經理人重新上傳

視窗下方顯示該單位的歷次重傳紀錄。

### 4.5 失敗通知管理（FinanceAdmin）

FinanceAdmin 可在儀表板下方看到 **「失敗通知」** 面板，顯示發送失敗的 Email 通知。
可輸入正確的收件人電郵後點擊 **「重送」** 重新發送。

### 4.6 CompanyReviewer 特殊行為

CompanyReviewer 在儀表板不會看到個別單位狀態，而是顯示提示訊息引導前往彙整報表頁面。

## 5. 彙整報表

**路徑：** 彙整報表  
**可用角色：** SystemAdmin、FinanceAdmin、UplineReviewer、CompanyReviewer

彙整報表整合部門營運預算、人力預算與公攤費用三大來源，呈現完整的預算彙整資料。

### 5.1 報表資訊

頁面頂部顯示：

| 資訊 | 說明 |
|------|------|
| 幣別 | 報帳幣別（由週期設定決定） |
| 預算更新 | 最近一次預算資料更新時間 |
| 人力更新 | 最近一次人力預算匯入時間 |
| 公攤更新 | 最近一次公攤費用匯入時間 |

### 5.2 報表欄位

| 欄位群組 | 欄位 | 說明 |
|----------|------|------|
| 基本資訊 | 單位代碼 / 單位名稱 | 組織單位識別 |
| 基本資訊 | 會科代碼 / 會科名稱 | 會計科目識別 |
| 基本資訊 | 實績 | 來自實績匯入的實際金額 |
| 部門營運預算 | 預算 | 部門提報的營運預算金額 |
| 部門營運預算 | 增減額 | 預算與實績的差額 |
| 部門營運預算 | 增減% | 增減百分比 |
| 人力預算 | 人力預算 | 由人資匯入的人力預算金額 |
| 公攤費用 | 公攤費用 | 由財務匯入的公攤費用金額 |

表格支援排序及分頁。

### 5.3 匯出報表

點擊 **「匯出」** 按鈕，選擇匯出格式：

- **匯出 Excel**：下載 .xlsx 檔案
- **匯出 CSV**：下載 .csv 檔案

資料量較小時會立即下載；資料量較大時系統會啟動非同步匯出，頁面顯示
**「正在產生報表...」** 進度提示，完成後自動下載。

## 6. 上傳預算

**路徑：** 上傳預算  
**可用角色：** FilingUnitManager

部門經理人透過此頁面下載預算樣板、填寫並上傳預算資料。

### 6.1 操作步驟

1. 系統自動選取目前開放中的預算週期
2. 點擊 **「下載樣板」** 取得 Excel 預算樣板
3. 在 Excel 中填寫預算資料
4. 點擊 **「選擇檔案」** 選取已填寫的 .xlsx 檔案
5. 系統自動上傳並驗證檔案
6. 上傳成功後顯示 **「上傳成功」** 通知

### 6.2 限制

- 僅接受 **.xlsx** 格式
- 檔案大小上限 **10 MB**
- 每次上傳產生新版本，不覆蓋舊版本

### 6.3 上傳紀錄

頁面下方顯示歷次上傳紀錄：

| 欄位 | 說明 |
|------|------|
| 版本 | 版本編號（遞增） |
| 上傳時間 | 上傳時間戳記 |
| 資料筆數 | 檔案中的資料列數 |
| 檔案大小 | 檔案位元組數 |
| 狀態 | 未上傳 / 已上傳 / 已通知重傳 |

### 6.4 收到重傳通知

若 FinanceAdmin 或 UplineReviewer 要求重傳，經理人會收到 Email 通知。
重新上傳後，狀態會更新為「已上傳」。

## 9. 週期管理

**路徑：** 週期管理  
**可用角色：** FinanceAdmin、SystemAdmin

管理預算週期的完整生命週期：建立、開放、關閉、重開。

### 9.1 週期狀態

```
草稿 (Draft) → 已開放 (Open) → 已關閉 (Closed)
                                       ↓
                                 重開 (Reopen) → 已開放 (Open)
```

| 狀態 | 說明 |
|------|------|
| 草稿 | 新建週期的初始狀態，尚未產生樣板 |
| 已開放 | 已產生樣板並通知經理人，可接受預算上傳 |
| 已關閉 | 週期結束，不再接受上傳 |

### 9.2 建立週期

1. 點擊 **「建立週期」** 按鈕
2. 填寫：
   - **財政年度**：例如 2026
   - **截止日期**：格式 YYYY-MM-DD
   - **報帳幣別**：例如 TWD
3. 點擊 **「確認」**

### 9.3 開放週期

1. 在草稿週期的卡片中點擊 **「開放週期」**
2. 系統執行以下自動作業：
   - 為每個填報單位產生 Excel 預算樣板
   - 向各單位經理人發送 Email 通知
3. 顯示結果摘要：
   - **樣板產生結果**：成功數 / 失敗數
   - **通知發送結果**：已送出 / 失敗數
4. 若有樣板產生失敗，可點擊 **「重試」** 單獨重新產生

> 開放前，請先確認「填報單位管理員確認」區塊中所有單位都已指定經理人。

### 9.4 設定提醒排程

1. 點擊 **「設定提醒排程」**
2. 在 **「提前提醒天數」** 欄位輸入正整數，以逗號分隔
   - 例如：`7, 3, 1` 表示截止日前 7 天、3 天、1 天各發送一次提醒
3. 點擊 **「儲存提醒設定」**

### 9.5 關閉週期

1. 在已開放週期的卡片中點擊 **「關閉週期」**
2. 確認對話框中點擊 **「確認」**
3. 關閉後將不接受新的預算上傳

### 9.6 重開週期（僅 SystemAdmin）

1. 在已關閉週期的卡片中點擊 **「重開週期」**
2. 填寫 **重開原因**（必填）
3. 點擊 **「確認」**

> 重開功能有時間窗口限制（預設關閉後 30 天內），超過期限無法重開。

## 10. 會科主檔

**路徑：** 會科主檔  
**可用角色：** FinanceAdmin、SystemAdmin（新增僅限 SystemAdmin）

維護系統中的會計科目 (Chart of Accounts)。

### 10.1 檢視帳號

- 表格列出所有會計科目，可依 **類別** 篩選：
  - 營運帳號
  - 人力帳號
  - 公攤帳號
- 每頁顯示 50 筆，支援分頁

| 欄位 | 說明 |
|------|------|
| 帳號代碼 | 會計科目代碼 |
| 帳號名稱 | 會計科目名稱 |
| 類別 | 營運 / 人力 / 公攤 |
| 層級 | 階層深度 |

### 10.2 新增帳號（SystemAdmin）

1. 點擊 **「新增帳號」**
2. 填寫：
   - **帳號代碼**（必填）
   - **帳號名稱**（必填）
   - **類別**（下拉選擇）
   - **層級**（正整數）
3. 點擊 **「儲存」**

> 若帳號代碼已存在，系統會更新該帳號資訊。

### 10.3 匯入實績（SystemAdmin）

1. 點擊 **「匯入實績」**
2. 輸入 **週期 ID**
3. 選擇實績檔案（.csv / .xlsx / .xls）
4. 點擊 **「匯入」**

## 11. 組織架構

**路徑：** 組織架構  
**可用角色：** SystemAdmin（完整權限）、FinanceAdmin（唯讀）

管理組織單位結構及填報單位設定。

### 11.1 組織單位表格

| 欄位 | 說明 |
|------|------|
| 代碼 | 組織單位代碼 |
| 名稱 | 組織單位名稱 |
| 層級代碼 | 組織層級標識 |
| 申報單位 | 是否為填報單位（是/否） |
| 有經理人 | 是否已指派經理人（無經理人時顯示警示圖示） |
| 排除週期 | 已排除的週期數量 |
| 操作 | 編輯按鈕（僅 SystemAdmin） |

### 11.2 編輯組織單位（SystemAdmin）

1. 點擊目標單位的 **「編輯」** 按鈕
2. 在彈出視窗中修改 **「排除週期」**（多選下拉，可選擇要排除該單位的週期）
3. 點擊 **「儲存」**

> 被排除的單位在該週期中不會出現在填報狀態中，也不會產生預算樣板。

## 12. 使用者管理

**路徑：** 使用者管理  
**可用角色：** SystemAdmin

管理系統使用者帳號、角色指派與帳號停用。

### 12.1 使用者表格

| 欄位 | 說明 |
|------|------|
| 名稱 | 使用者顯示名稱 |
| 電郵 | 使用者電子郵件 |
| 角色 | 已指派的角色標籤 |
| 狀態 | 啟用（綠色）/ 停用（紅色） |
| 操作 | 編輯角色 / 停用帳號 |

### 12.2 編輯角色

1. 點擊目標使用者的 **「編輯角色」** 按鈕
2. 在彈出視窗中勾選或取消角色：
   - FilingUnitManager（申報單位經理人）
   - HRAdmin（人資管理員）
   - FinanceAdmin（財務管理員）
   - UplineReviewer（上級審核人）
   - CompanyReviewer（公司審核人）
   - ITSecurityAuditor（資安稽核人員）
   - SystemAdmin（系統管理員）
3. 點擊 **「儲存」**

### 12.3 停用帳號

1. 點擊目標使用者的 **「停用帳號」** 按鈕
2. 確認對話框提示：此操作無法撤銷
3. 點擊紅色 **「停用帳號」** 按鈕確認

> **注意：** 帳號停用後無法恢復。已停用的帳號無法再登入系統。

## 13. 稽核日誌

**路徑：** 稽核日誌  
**可用角色：** ITSecurityAuditor、SystemAdmin

查詢系統所有操作紀錄，驗證稽核鏈完整性。

### 13.1 篩選條件

| 篩選項目 | 說明 |
|----------|------|
| 使用者 ID | 依操作者 ID 篩選 |
| 操作 | 選擇操作類型（如 LOGIN_SUCCESS、BUDGET_UPLOADED 等） |
| 資源類型 | 依資源類型篩選（如 cycle、user、budget_upload） |
| 資源 ID | 依特定資源 ID 篩選 |
| 開始時間 | 查詢起始時間 |
| 結束時間 | 查詢結束時間 |

點擊 **「搜尋」** 執行查詢；點擊 **「取消」** 清除所有篩選條件。

### 13.2 稽核日誌表格

| 欄位 | 說明 |
|------|------|
| 時間 | 操作發生時間 |
| 使用者 ID | 執行操作的使用者 |
| 操作 | 操作類型 |
| 資源類型 | 被操作的資源類型 |
| 資源 ID | 被操作的資源 ID |
| IP 位址 | 操作者 IP 位址 |

每頁 50 筆，支援分頁。

### 13.3 驗證完整性

系統使用雜湊鏈 (Hash Chain) 確保稽核日誌不被竄改。

1. 設定 **開始時間** 與 **結束時間**
2. 點擊 **「驗證完整性」**
3. 彈出視窗顯示：
   - **驗證結果**：完整（綠色）或 已損毀（紅色）
   - **鏈長度**：該範圍內的紀錄筆數

### 13.4 匯出 CSV

1. 設定開始時間與結束時間
2. 點擊 **「匯出 CSV」**
3. 瀏覽器自動下載 `audit_logs.csv` 檔案

## 14. 常見問題

### 登入相關

**Q：登入後看到「無存取權限」**  
A：您的帳號尚未被指派角色。請聯繫系統管理員 (SystemAdmin) 指派適當角色。

**Q：操作到一半被登出**  
A：系統閒置超過 30 分鐘或登入超過 8 小時會自動登出。重新登入後即可繼續操作。

### 預算上傳

**Q：顯示「目前無開放中的預算週期」**  
A：目前沒有狀態為「已開放」的週期。請聯繫財務管理員確認週期是否已開放。

**Q：下載樣板時顯示「此單位的預算樣板尚未產生」**  
A：週期尚未正式開放或樣板產生失敗。請聯繫財務管理員重新開放週期或重試樣板產生。

**Q：上傳時顯示「檔案大小超過 10 MB 限制」**  
A：請檢查檔案是否包含不必要的工作表或圖片，精簡後重新上傳。

**Q：上傳時顯示「僅接受 .xlsx 格式檔案」**  
A：系統僅接受 Excel 2007+ 格式 (.xlsx)。若您的檔案為 .xls 格式，請先另存為 .xlsx。

**Q：收到重傳通知 Email**  
A：財務管理員或上級審核人要求您重新提交預算。請修正後重新上傳，新版本會自動產生。

### 報表與匯出

**Q：彙整報表顯示「此週期尚無彙整資料」**  
A：該週期尚未有任何預算上傳、人力匯入或公攤匯入。資料匯入後即可檢視報表。

**Q：匯出等待很久**  
A：資料量較大時系統會啟動非同步匯出。頁面會顯示進度提示，完成後自動下載。請勿關閉頁面。

### 週期管理

**Q：開放週期後部分樣板產生失敗**  
A：結果摘要會列出失敗的單位及原因。可點擊「重試」按鈕單獨重新產生失敗的樣板。

**Q：無法重開已關閉的週期**  
A：僅 SystemAdmin 可重開週期，且需在關閉後 30 天內。超過期限請聯繫系統管理員評估處理方式。

### 其他

**Q：在組織架構頁面無法編輯**  
A：FinanceAdmin 僅有唯讀權限。編輯組織架構需 SystemAdmin 角色。

**Q：停用的帳號可以恢復嗎？**  
A：帳號停用後無法恢復。如需重新啟用，需由系統管理員另行處理。
