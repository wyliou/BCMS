# Spec: i18n Setup (simple)

Module: `frontend/src/i18n/index.ts` + `frontend/src/i18n/zh-TW.json`
Tests: `frontend/tests/unit/i18n/i18n.test.ts`

## Exports
- `i18n` — configured `i18next` instance (default export from `index.ts`).

## Imports
- `i18next`: `i18n` instance
- `react-i18next`: `initReactI18next`
- `./zh-TW.json`: translation resource

## Setup Requirements

**`index.ts`**
- Call `i18n.use(initReactI18next).init({...})` with:
  - `lng: 'zh-TW'` (primary language)
  - `fallbackLng: 'zh-TW'` (no English fallback; missing keys show key string)
  - `resources: { 'zh-TW': { translation: zhTW } }`
  - `interpolation: { escapeValue: false }` (React already escapes)
  - `debug: false` (never true in production)

**`zh-TW.json`** — Seed keys required for Batch 7 modules. Key format: `<page>.<section>.<key>`.

## Side-Effects
- `i18n.init()` is called as a module-level side effect when `index.ts` is first imported. Consumers must import this module before rendering any component that uses `useTranslation()`.

## Gotchas
- `index.ts` must be imported in `App.tsx` (or `main.tsx`) before the React tree is rendered.
- Do NOT use `await i18n.init()` in module scope; `initReactI18next` handles the async init with a React Suspense fallback.
- Never hardcode Chinese characters in `.tsx` files. All user-facing strings go in `zh-TW.json`.
- English is used for: role enum values (`FinanceAdmin`, `HRAdmin`, etc.), API field names, error codes (`UPLOAD_007`), technical labels.
- i18n key format: `<page>.<section>.<key>` (e.g., `dashboard.summary.total_units`).

## Tests
1. `i18n.t('common.loading')` returns `'載入中...'` after initialization.
2. `i18n.language` equals `'zh-TW'` by default.
3. Missing key returns the key string (not throws).
4. `i18n.t('status.not_uploaded')` returns `'未上傳'`.
5. `i18n.t('nav.dashboard')` returns `'儀表板'`.

## Constraints
FCR-004: All user-visible strings in this component are rendered via `t('i18n.key')`. No inline Chinese characters in JSX.
